# DPO v6: Minimal Silence 計画概要

SFT v3 の失敗（データ汚染と不安定性）を教訓に、最も高い実績（スコア0.75）を持つベースモデルを最小限の労力で「沈黙」させるための最終フェーズです。

## 1. 狙い (Goal)
- **思考過程（Chain of Thought）の完全抑制**: 「Approach: ...」や挨拶文の徹底排除。
- **Markdownフォーマットの抑制**: 評価時に減点対象となる ```json などのバッククォート記法を根絶。
- **モデルの安定性維持**: 最小限の学習（1エポック、低学習率）により、SFT v3 で発生した「無限ループ」や「幻覚」を回避。

## 2. 手法 (Method)

### データセット構成
- **ソース**: `structured-hard-sft-4k.jsonl` （多様なJSON/XML/CSV/YAMLタスクを含む）
- **サンプル数**: 300件（ターゲットを絞った最小構成）
- **ペア作成ロジック**:
    - **Chosen (正解)**: 元データから思考過程、挨拶文、Markdown記法をプログラムで徹底的に除去した「純粋な出力」。
    - **Rejected (不正解)**: 
        - 元データに思考過程が含まれる場合はそのまま採用。
        - 綺麗なデータの場合は、あえて「Here is the result...」や「```json」を付加して汚したものを作成。
    - **狙い**: 「答えが正しくても、余計なことを喋ったりMarkdownを使ったりすると評価が下がる」ことを明確に教え込む。

### 学習設定
- **ベースモデル**: `Qwen3-4B-Instruct-2507` + `adapter_legacy_sft_20260209_0709` (スコア0.75の実績)
- **手法**: PEFT (LoRA) + DPO (Direct Preference Optimization)
- **学習率**: `1e-6` (モデルの知識を壊さないよう極めて低く設定)
- **エポック**: `1` (約75ステップ)
- **マージ戦略**: SFTアダプタをマージした状態でDPOを適用し、スタイル（話し方）のみを矯正。

## 3. 期待される効果
- **スコア向上**: フォーマット違反（Markdown、挨拶）による減点をゼロにし、0.8以上のスコアを安定して出す。
- **推論速度の向上**: 余計な思考や挨拶、ループがなくなることで、トークン生成効率が最大化。
- **一貫性**: どのようなフォーマット（CSV, YAML含む）の指示に対しても、常に純粋なデータのみを返す一貫性の獲得。
