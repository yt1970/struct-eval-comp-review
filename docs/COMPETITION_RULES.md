# コンペティションの厳密なルールと制約

このドキュメントは、本コンペティションにおいて**絶対に遵守しなければならないルールと制約**をまとめたものです。ジェミサン（AI）の全行動・戦略は、このルールを前提に構築されます。

## 1. 許容されるデータソース
- **使用可能な外部データ**: `/Users/yutako/dev/struct-eval-comp/data/hf_datasets` に配置されているデータセット**のみ**使用可能。
- **データの加工**: 許可されているデータセットを**プログラム的・機械的に加工（スクリプトによる文字列操作、正規表現、BERT等による抽出・変形など）することは許可されている**。

## 2. 禁止されているデータ生成手法 (**重大な違反**)
- **LLMを用いたデータ生成**:
  - Qwenやその他のいかなるLLM（ChatGPT, Claude等含む）を使用して、学習用の合成データ（プロンプトや期待される回答など）を新たに生成することは**完全に禁止**されている。
  - いわゆる「LLM-as-a-Judge」や「LLMによるデータ拡張（Data Augmentation）」は行えない。
- **本番クエリの流用**:
  - `public_150.json` (本番の評価用データ) の `query` の文章を、学習データとしてそのまま利用したり、訓練に組み込んだりすることは**禁止**。

## 3. プロンプトエンジニアリングの禁止
- **推論時のプロンプト改変禁止**:
  - 本番の推論時（評価時）において、運営から与えられる `query` に対して、システムプロンプトを追加したり、「Markdownを使わないで」といった指示文を付け加えたりする等の**プロンプトエンジニアリングは一切禁止**されている。
  - アダプター適用後のモデルは、運営から与えられた素の `query` だけで、望ましい沈黙の出力（フォーマット遵守）を行えなければならない。

## 4. 許可されているモデル/ツール
- **基本モデル**: `Qwen/Qwen3-4B-Instruct-2507` (固定)
- **許容される補助ツール**: **BERT** などの非生成系言語モデルは、データの分類、クラスタリング、特徴抽出、あるいはルールベースの加工補助などの目的で使用可能。
