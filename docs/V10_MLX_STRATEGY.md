# V10 Strategic Plan: MLX Silent Generalization

## 1. 狙い（Objective）
V9において発生した「生クエリ（Raw Query）に対するお喋り（Chatty）化」を根本から解決し、**「どんな形式の指示が来ても、沈黙して純粋なコードのみを出力する」**強靭なモデルを構築する。
DPOでの対照学習ではなく、MLXを用いたLoRA（SFT）により、出力形式の「正解」を強力に刷り込む。

## 2. 手法（Methodology）
### 機械的沈黙データセットの構築
- **ソースデータ**: `hf_datasets` の約1.3万件を使用。
- **クリーニング**: `Approach:` や `Output:` などの不要な解説テキスト、およびMarkdownのコードブロック記号 (```) を正規表現で完全に除去。
- **形式**: `messages` 形式で、`assistant` 側には純粋なコード（JSON, XML, TOML等）のみを格納。
- **多様性**: 入力クエリはそのまま残しつつ、出力側の「解説なし」という一貫性を重視。

### MLX-LM による高速学習
- Apple Silicon (M3 Mac) に最適化された `mlx-lm` を利用。
- 量子化（4bitなど）の恩恵を受けつつ、効率的にLoRAアダプタを学習。

## 3. 計画（Execution Plan）
1.  **データ準備**: `scripts/v10_mlx/prepare_data.py` で 1.2万件の精鋭セットを作成。（完了）
2.  **MLX学習**: `mlx_lm.lora` を用いて、1000イテレーションの集中的な「沈黙特訓」を実施。
3.  **マージ**: 学習したアダプタをベースモデルに統合。
4.  **最終検証**: `verify_v9_full_150.py` を改修し、マージ後のモデルでスコアと出力形式をダブルチェック。
5.  **アップロード**: 納得の結果が得られたら Hugging Face へ最終モデルとしてアップロード。

| 項目 | 設定値 | 備考 |

## 5. 検証計画（Validation Plan）
- **定量的評価**: `public_150.json` に対する精度チェック（目標: スコア 0.8以上）。
- **定性的評価**: 
    - 冒頭に挨拶がないか？
    - 末尾に余計な解説がないか？
    - ```json などの Markdown 記号が漏れていないか？
    - 素のクエリ（Raw Query）に対し、余計な「Here is it...」を言わないか。

## 6. 期待成果（Expected Outcome）
- どのようなプロンプト形式であっても、出力の先頭と末尾が完全に「コードのみ」であること。
- 余計な解説部分を削ることで、推論速度の向上と正確な期待フォーマットへの一致を実現。
- コンペルールを100%遵守した「クリーンな構造化データ生成器」の完成。
