# V13 Strategy: The Silent Brain (沈黙する知能)

## 1. 目的 (Objective)
DPO2で得られた「圧倒的な構造化精度」を、0.75 SFTベースモデルの「極めて高い静粛性」の上に完全移植し、0.80の壁を突破する。

## 2. 狙い (Goal)
- **知能の継承**: DPO2 (0.73 -> 0.76) で成功した「思考プロセス (Approach)」の有用性を活用する。
- **静粛性の極致**: 0.75ベースモデルの「余計なことを言わない」性質を維持、あるいはさらに強化する。
- **内面化された推論**: 思考はプロセス（脳内）で完結させ、出力（発話）は構造化データのみに限定する。

## 3. 前回 (V12) の教訓 (Lessons Learned)
- **拒絶反応**: 0.75（無口モデル）に対し「Approachを詳細に書け」というDPO2レシピをそのまま適用した結果、モデルが混乱しチャタネスが80%まで悪化。
- **学習の不安定さ**: MPS環境での全パラメータLoRA＋高い学習率（5e-6）により `grad_norm` が NaN になり、モデルの崩壊を招いた。
- **方針転換**: 沈黙モデルに無理にお喋りを教えるのではなく、「考えてから黙って出す」という一段上の挙動を教える。

## 4. データ作成の方針 (Data Strategy)
DPO2のソースである `u-10bei/dpo-dataset-qwen-cot` をベースに、外科的な加工を行う。

| 項目 | 設定内容 |
|------|----------|
| **ソース** | `u-10bei/dpo-dataset-qwen-cot` (DPO2の出典) |
| **配合** | 構造化データ (JSON/CSV/YAML) 150件 + 一般能力維持用 100件 = 計250件 |
| **Chosen (正解)** | 元の `chosen` から **`Approach:` セクションを完全に削除**し、`Output:` 以降の構造化データのみを残す。 |
| **Rejected (不正解)** | 元の `rejected` または `Approach` セクションが含まれる饒舌な回答。 |

## 5. 学習の方針 (Training Policy)
- **Base Model**: 0.75 SFT版 (4000件学習済み)
- **Learning Rate**: `2e-6` (DPO3で安定した超微調整設定)
- **LoRA Config**: r=16, alpha=16, All-7-modules
- **Precision**: bfloat16 / fp16 (MPSの安定性を考慮)
- **Implicit Reasoning**: プロンプトで「Approachを書いてからOutputを出せ」と指示しつつ、Chosenを「Outputのみ」にすることで、出力には出さないが内部で思考ステップを踏むよう誘導する。
