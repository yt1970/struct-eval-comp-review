# 🚀 LLMコンペ「0.8突破」本気学習戦略レポート

**作成日:** 2026年2月3日  
**ターゲット:** `Qwen/Qwen3-4B-Instruct-2507`  
**目標:** パースエラーを物理的に排除し、高難度な構造化タスクを完遂する。

---

## 1. 実行環境（Hardware & Software）
Mac M4チップの性能を最大限に引き出しつつ、提出時の互換性を最優先した構成です。

*   **Hardware:** Apple M4 Chip (MPS: Metal Performance Shaders をフル活用)
*   **Precision:** `bfloat16` (Mac M4において精度と速度のバランスが最も安定)
*   **Library:**
    *   `PyTorch` (MPS Direct ML)
    *   `Transformers` (5.0.0rc3)
    *   `PEFT` (標準LORA形式でのアダプタ出力用)
    *   `TRL` (`SFTTrainer` による効率的な学習)

---

## 2. データ拡張戦略（BERT-Augmentation）
規約で許可されている「LLM以外」の手法を用い、データの純度を保ちつつ量を3倍に増幅します。

*   **手法:** BERTによる単語置換（`ContextualWordEmbsAug` 相当の直書き実装）
*   **対象:** `structured-hard-sft-4k.jsonl` (4,000件)
*   **プロセス:** 
    1.  人間の質問（`user`）の中からランダムに単語を選び、それを `[MASK]`（穴埋め）に変換。
    2.  BERT（`bert-base-multilingual-cased`）にその穴を埋めさせ、文脈に合った別の表現を生成。
    3.  1つのデータに対し2つのバリエーションを作成し、合計 **12,000件** の強固なデータセットに。
*   **狙い:** 指示のわずかな揺れ（「JSONにして」→「JSONへ変換」など）に左右されない安定した回答を学習させる。

---

## 3. 「型」の徹底固定（Output Cleaning）
パースエラー（0点）を物理的に防ぐための、本戦略の最重要ポイントです。

*   **前処理:** 学習実行プログラム内で、AIの回答（`assistant`）をリアルタイムでクリーニング。
*   **ロジック:** 回答テキストの先頭をスキャンし、**最初に出現する `{` (JSON) または `<` (XML) よりも前の文字列をすべて削除** する。
*   **狙い:** 「はい、承知しました」といった前置き（枕詞）をモデルに一切学習させない。これにより、推論時にも「いきなり記号から書き始める」挙動を強制し、正規表現パースの成功率を100%に近づける。

---

## 4. ハイパーパラメータ（SFT/LoRA 設定）
一日の終わりに「放置学習」で最高の「型」を焼き上げるための設定です。

| パラメータ | 設定値 | 狙い |
| :--- | :--- | :--- |
| **Max Steps** | 1000 | 12,000件のデータをじっくりと読み込ませる |
| **Learning Rate** | `5e-5` | 指示通りの最適な学習率 |
| **LR Scheduler** | `cosine` | 学習終了間際に微調整を行い、精度を極める |
| **Warmup Ratio** | `0.1` | 学習初期の不安定な動きを抑制する |
| **Batch Size (GA)** | `1 (×8)` | M4のメモリを節約しつつ、実質バッチ8で安定させる |
| **Target Modules** | 全主要パーツ* | q, v だけでなく gate, up, down 等も特訓対象に |
| **LoRA R / Alpha** | `16 / 32` | 表現力が不足しないよう、少し厚めの設定 |

*\*target_modules: ["q_proj", "v_proj", "k_proj", "o_proj", "gate_proj", "up_proj", "down_proj"]*
