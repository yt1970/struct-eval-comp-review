# V11 Surgical DPO 戦略レポート：沈黙の知能（Silent Intelligence）の完成

## 1. V11 の狙い：知能と沈黙の「真の融合」
これまでの試行錯誤（DPO4～11）の反省から、単一モデルへの強制的な学習ではなく、**「既に賢い脳（DPO3）」** と **「既に沈黙をマスターしている脳（V10）」** を外科的に結合し、その不安定な合体状態を DPO で「賢くて静かな状態」へと調律（アライメント）します。

## 2. これまでの反省と教訓
- **知能崩壊への恐怖**: 沈黙データのみを与え続けると、語彙が極端に制限され、推論能力が落ちる（V10の教訓）。
- **お喋りの残存**: 知能を残そうとすると、Instructモデル特有のサービス精神（Preamble）が消えきらない。
- **結論**: 両方の性質を持つモデルを物理的にマージした状態を「スタート地点」にし、DPOを「不要なノイズ（おバカ成分）を削ぎ落とす彫刻刀」として使う。

## 3. 学習手法：二重外科マージ ＆ 標的型DPO
### ステップ1：物理的モデル融合（Double-Merge）
以下の2つの異なる進化を遂げた脳を、ベースモデルにマージします。
1. **知能の脳 (DPO3)**: 0.77064 を記録した最高 IQ モデル。
2. **沈黙のプロトタイプ (V10 Iter 100)**: 知能は低いが、徹底的に「お喋り」が封じ込められたモデル。
これを合体させ、**「賢いけれど、黙る癖も細胞レベルで知っている」** ベースモデルを構築します。

### ステップ2：標的型 LoRA（Targeted LoRA）
学習対象を `q_proj` と `v_proj` のみに限定します。これはモデルの「知識」ではなく「どこに注目して出力するか（Attention）」の振る舞いだけを矯正し、0.77の知能を無傷で残すためです。

## 4. 学習パラメータ
| パラメータ | 設定値 | 理由 |
| :--- | :--- | :--- |
| **Max Steps** | **50 steps** | 長時間の学習は「知能崩壊」の元。極短時間で癖だけを直す。 |
| **Learning Rate** | **2e-6** | V9で沈黙化に成功した実績値。5e-7では弱すぎた。 |
| **Batch Size** | 1 (Grad Accum: 8) | Mac M4 での安定性と精度を両立。 |
| **Beta** | 0.1 | 参照モデルからの乖離を抑える標準設定。 |
| **Precision** | **bfloat16** | M4チップの性能を最大化する。 |

## 5. DPOデータ生成戦略（合法・高精度）
- **Chosen (沈黙)**: `hf_datasets` の正解コードから、正規表現で 100% 確実に Markdown と挨拶を除去。
- **Rejected (お喋り)**: 0.77モデルが吐いた失敗パターン（`Sure!`, ` ```json `）をアルゴリズムで正確に模倣し、合成。
- **ChatML統一**: `apply_chat_template` を学習時・推論時で完全に一致させ、形式のズレによるお喋り誘発を封じ込める。

## 6. 期待される効果
1. **0.80の壁突破**: お喋りによるパースエラーで削られていた 5～10% のスコアを回収。
2. **構造破綻ゼロ**: 最小限の LoRA ターゲットにより、ネストした JSON や複雑な XML の破壊を防止。
3. **レギュレーション遵守**: 生成系LLMや本番クエリを使わない、完全に合法な手法での勝利。
